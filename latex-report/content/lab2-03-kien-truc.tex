\section{Kiến Trúc Tổng Quan}

Hệ thống được thiết kế theo mô hình split-repo: app repo xử lý source code,
quality gates và Docker image; CD repo lưu desired state để ArgoCD đồng bộ vào
Kubernetes. Cách tách này giúp Jenkins không trực tiếp thay đổi workload trên
cluster, đồng thời mọi thay đổi deploy đều có lịch sử Git rõ ràng.

\subsection{Luồng End-to-End}

\begin{lstlisting}[language=bash, caption={Luồng source-to-runtime của đồ án}]
Developer push branch/tag -> Jenkins app repo pipeline
  -> Gitleaks/Test/Coverage/SonarQube gate
  -> Docker Hub image docker.io/<user>/yas-<service>:<tag>
  -> Jenkins commits overlay change to yas-cd/main
  -> ArgoCD detects Git change
  -> K3s reconciles dev/staging/developer namespaces
\end{lstlisting}

\subsection{Hạ Tầng Triển Khai}

Hạ tầng thực tế của nhóm tách rõ CI và CD/runtime. Jenkins Controller chạy trên
AWS EC2 tại địa chỉ \texttt{3.27.92.213}; các bước CI nặng trong Jenkinsfile
chạy trên ba laptop agent \texttt{Agent-Tri}, \texttt{Agent-QVinh},
\texttt{Agent-TVinh} với label \texttt{yas-build-worker}. Máy GCP
\texttt{gcp-ci-cd-agent} không dùng làm Jenkins build agent trong kiến trúc
báo cáo; máy này dành cho CD/runtime, gồm K3s single-node, ArgoCD, ingress,
service mesh và workload YAS. Cấu hình VM GCP là 40 GB RAM và 150 GB
persistent disk.

Ở lớp network, Google Cloud firewall mở các port phục vụ demo và vận hành:
\texttt{8083} cho Jenkins UI, \texttt{6443} cho Kubernetes API server,
\texttt{30000-32767} cho NodePort/Ingress demo, \texttt{50000} cho Jenkins
inbound agent và \texttt{22} cho SSH quản trị. Các rule này là điều kiện để máy
ngoài có thể truy cập Jenkins, ArgoCD/app qua NodePort và để agent kết nối về
controller.

\subsection{Runtime Policy Trên Single-Node}

Do toàn bộ CD/runtime chạy trên một VM duy nhất, nhóm không giữ ba môi trường full-stack
cùng active. Baseline là:

\begin{itemize}
    \item \texttt{dev}: active, auto-sync.
    \item \texttt{staging}: active nhưng manual sync cho release approval.
    \item \texttt{developer}: dormant, chỉ bật khi chạy developer preview.
    \item Khi \texttt{developer} active, \texttt{staging} chuyển dormant để tránh quá tải single-node.
\end{itemize}
