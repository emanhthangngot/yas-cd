\subsection{Hai Repository Trong Phạm Vi Báo Cáo}

\begin{itemize}
    \item App/CI repo: \url{https://github.com/tzin1401/yas.git}. Repository này chứa source code YAS, Jenkinsfile chính, \texttt{services.yaml}, Dockerfile, CI scripts, test, và phần instrumentation cho Prometheus/OpenTelemetry.
    \item CD/GitOps repo: \url{https://github.com/emanhthangngot/yas-cd.git}. Repository này chứa desired state cho Kubernetes: \texttt{base/}, \texttt{overlays/}, \texttt{argocd/}, \texttt{charts/}, Jenkins job GitOps nhẹ và evidence.
\end{itemize}

Ranh giới quan trọng: app repo sinh ra image và trigger GitOps commit; CD repo là nguồn sự thật cho trạng thái cluster. Nhóm không dùng \texttt{kubectl set image} hay \texttt{kubectl apply} trực tiếp vào namespace do ArgoCD quản lý.
